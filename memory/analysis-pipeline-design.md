Pipeline order: extract â†’ `fillGaps` â†’ `smoothFrames` â†’ `normalizeAll` â†’ `computeAnglesSequence` â†’ handedness decision â†’ DTW (sequence) or best-frame match (single_frame) â†’ per-joint deltas â†’ phases.

Non-obvious constraints / decisions (added 2026-05-28):
- **Cannot run the real pipeline here** â€” no real video, no WASM/GPU runtime. The ONLY validation is unit tests on synthetic data + typecheck + build. Favor provably-correct-on-synthetic changes over anything needing empirical tuning. The four-command gate is `npm run test` / `npm run typecheck` / `npm run build`.
- **`emptyFrame()` zero-pose bug**: undetected frames used to inject 180Â° "fully extended" limb angles. `prepare.ts:fillGaps` interpolates per-landmark across time to fix this; measure `detectionCoverage` on RAW frames (before fill) for the confidence caveat.
- **Smoothing** (`smoothFrames`) uses a binomial filter, default radius 1, deliberately small so it does not blur the velocity peak `phases.ts` keys on.
- **Handedness** (`handedness.ts`): mirroring is a *column swap* on the 13-feature angle vector (not a 33-landmark mirror). Sequence mode infers dominant side from motion energy; single-frame mode (pro image) has no motion so it picks the orientation (raw vs mirrored user angles) with the smaller best-frame match distance â€” do NOT hard-default the pro side to 'right' (that regresses lefty-user cases).
- **`hip_rotation` was removed** â€” it was hip-line vs world X-axis, the only camera-yaw-dependent feature, meaningless across two differently-filmed clips. Replaced by `trunk_lean` (torso axis vs vertical/gravity), which is yaw-invariant. All 13 features are now either intra-body relative angles or measured against gravity.
- `handedness` and `coverage` on `AnalysisReport` are optional for backward-compat with previously-saved SQLite records.

Coaching guide + workouts are now generated NATIVELY (no API key, no network) â€” added 2026-05-30:
- Engine lives at `renderer/src/lib/coach.ts` (deterministic rule engine, KB keyed by joint group + sign + magnitude). It imports `@shared/types`. That's why the engine lives in the renderer tree with `@shared/types`, not in `shared/`. Keep renderer-only modules importing `@shared/types`; don't put a vite-imported module in `shared/` with relative sibling imports.
